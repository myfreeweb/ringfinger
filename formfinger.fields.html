formfinger.fields documentation

All functions in this module make fields for you to use
with restfinger.core.
They're defined as functions even when they need no
customization just for consistency.

alphanumeric

(alphanumeric)
Validates alphanumeric strings

checkbox

(checkbox)
A boolean field, input type=checkbox

color-field

(color-field)
Validates hexadecimal color codes, input type=color

date-field

(date-field)
Validates/parses/outputs dates, input type=date

date-time-field

(date-time-field)(date-time-field local)
Validates/parses/outputs date+times, input type=datetime

double-field

(double-field)(double-field step)
Validates/parses double numbers, input type=number

double-range-field

(double-range-field)(double-range-field step)
Validates/parses double numbers, input type=range

email-field

(email-field)
Validates email addresses, input type=email

email-lookup

(email-lookup)
Validates email addresses with an additional DNS lookup. Safer, but slower, input type=email

ipv4-field

(ipv4-field)
Validates IPv4 addresses

maxlength

(maxlength n)
Sets the maximum length to the given number

minlength

(minlength n)
Sets the minimum length to the given number

nbetween

(nbetween minn maxn)
Sets the minimum and maximum numbers to given ones

nmax

(nmax n)
Sets the maximum number to the given one

nmin

(nmin n)
Sets the minimum number to the given one

non-confusing

(non-confusing)
Validates strings for confusing Unicode characters
(the ones that look the same in different scripts)

not-in

(not-in lst)
Given a list of forbidden values, does the right thing.
Useful if you have routes like /about and /:username
and you don't want someone to have an username 'about'
